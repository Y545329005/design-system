# UI Review Checklist

## 页面布局

- [ ] 信息层级是否清晰
- [ ] 留白是否合理
- [ ] 是否遵循统一布局
- [ ] 首页/落地页是否无侧边栏（原则七）
- [ ] 信息密度是否从 Hero 向下方递进（原则六）

---

## Typography

- [ ] 字号是否统一
- [ ] 行高是否舒适
- [ ] 层级是否清晰
- [ ] 是否仅使用单一字体族（原则四）
- [ ] 字重是否不超过 4 种（400/500/600/700）
- [ ] 标题层级是否 ≤ 3 级

---

## 配色

- [ ] 是否遵循设计规范
- [ ] 是否存在过多颜色
- [ ] 对比度是否合理
- [ ] 是否仅使用一个强调色（原则一）
- [ ] 正文是否未使用纯黑 `#000`（原则二）
- [ ] 背景是否未使用纯白 `#FFF`（原则二）
- [ ] 语义色是否仅用于状态传达（红/绿/黄不用于装饰）
- [ ] 是否优先使用透明度方案（black/5、gray-950/8）而非固定色值

---

## Elevation（层次）

- [ ] 是否使用了 `box-shadow`？应替换为 `inset-ring` + 背景明度差（原则三）
- [ ] 卡片层次是否通过背景色明度差传达（而非阴影）
- [ ] 描边是否使用 `inset-ring` 而非 `border`（不占布局空间）
- [ ] 暗色模式下 `inset-ring` 是否使用 `white/10` 等透明度反转
- [ ] 分区卡片的内部隔线是否使用 `border-gray-950/5`（最轻透明度）

---

## 组件

- [ ] 是否全部复用已有组件
- [ ] 是否存在重复设计
- [ ] 是否保持一致性
- [ ] 交互元素圆角是否统一 `rounded-lg`（不混用 `rounded-full`）
- [ ] 卡片内按钮是否引用 Button 组件样式（不自行定义）
- [ ] 容器圆角是否统一 `rounded-xl`

---

## 命名

- [ ] 组件名是否描述了"是什么"而非"从哪来"（不以数据源命名）
- [ ] 组件名能容纳未来 3 年的扩展吗（不以单一能力命名）
- [ ] 是否需要命名空间（领域可能包含多个子模块时）
- [ ] 变体名是否描述使用场景或视觉风格（非技术实现细节）
- [ ] 是否有缩写（永远用全称——Button 而非 Btn）

---

## 动画

- [ ] 过渡时长是否 ≤ 200ms（原则五）
- [ ] 是否仅过渡 `color` / `transform` / `opacity`（不过渡 `width`/`height`）
- [ ] 是否添加 `motion-reduce:transition-none`
- [ ] Press 反馈是否使用 `active:scale-[0.97]`（按钮）/ `active:scale-[0.99]`（卡片）
- [ ] 图片 hover 缩放是否 ≤ 3%

---

## 交互

- [ ] Hover 是否自然
- [ ] Focus 是否完整（`focus-visible:outline-2 outline-offset-2`）
- [ ] Loading 是否完善（保持宽度 + `cursor-wait` + 进行时文案）
- [ ] 空状态是否设计
- [ ] 纯图标按钮是否提供 `aria-label`

---

## 响应式

- [ ] PC（≥ 1280px）
- [ ] Pad（≥ 768px）
- [ ] Mobile（< 768px）
- [ ] 是否 Mobile First（默认样式为移动端）
- [ ] Grid 是否使用 `sm:grid-cols-2 lg:grid-cols-3` 渐进式列数

---

## 可访问性

- [ ] 颜色对比度是否 ≥ 4.5:1（WCAG AA）
- [ ] 图片是否有描述性 `alt`
- [ ] 是否使用语义 HTML（`<nav>`、`<main>`、`<article>`）
- [ ] 键盘导航是否完整（Tab / Enter / Space）
- [ ] Focus ring 是否未被移除
- [ ] 是否响应 `prefers-reduced-motion`

---

## 可维护性

- [ ] 是否容易扩展
- [ ] 是否容易复用
- [ ] 是否避免重复代码
- [ ] 新增设计是否值得沉淀（组件 / 模板 / Token / 原则）

---
