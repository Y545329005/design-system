# Stripe 官网设计分析

> 分析日期：2026-07-01
> 分析对象：stripe.com（官方网站及内部设计系统）
> 目的：提炼可复用的设计规律，而非复制

---

## 一、页面布局

### 整体结构

Stripe 官网采用**单页纵向滚动 + 深浅交替节奏**结构，从上到下依次为：

```
Header（固定导航）
  └─ sticky, backdrop-blur 毛玻璃
  └─ Logo + 极简导航链接 + CTA 按钮
  └─ 白色背景 + 细边框底部

Hero（主视觉）
  └─ 白色背景 + 极细重量标题
  └─ 大标题（weight 300，近乎"耳语"）
  └─ 副标题（slate 灰）
  └─ 紫色 Primary CTA + Ghost 次要按钮
  └─ 渐变光晕装饰（Ruby → Magenta）

Client Logos / 数据展示
  └─ "Global GDP running on Stripe" 实时数据
  └─ 信任背书数据（135+ currencies, $1.9T volume）

Feature Sections（产品区块）
  └─ 左右分栏：产品截图 Card + 功能文案
  └─ 白色区块与深色区块（#1c1e54）交替

Enterprise Customers（客户案例）
  └─ Accordion 展开式大客户展示
  └─ 图片 + 数据 + 产品使用标签

Code Demo（开发者展示）
  └─ 代码块 + 终端模拟

Testimonials（引用轮播）

CTA / Footer
  └─ 深色背景（#1c1e54）+ 白色文字
```

### 布局规律

| 特征 | 做法 | 原因 |
|---|---|---|
| **深浅交替节奏** | 白色 section → 深蓝 section → 白色 section | 创造戏剧性光暗节奏，避免单调 |
| **单栏纵向流** | 内容居中约束，自上而下自然阅读 | 降低认知负担 |
| **内容居中约束** | `max-width: 1080px` + `mx-auto` + gutter 32px | 大屏不散，视觉焦点集中 |
| **每屏一个主题** | 大标题 + 精简文案 + 产品截图卡片 | 单主题聚焦 |
| **信息密度递进** | Hero 极少 → 往下逐渐增加细节 | 先吸引，再说服 |
| **产品截图嵌入卡片** | 使用蓝调多层阴影的卡片展示产品 UI | 让产品看起来像漂浮在页面上 |

### App Shell（Stripe Dashboard）

Stripe 控制台采用**侧边栏 + 内容区**的经典布局：

```
┌──────────┬──────────────────────────┐
│ Sidebar  │  Top Bar                  │
│          ├──────────────────────────┤
│ 深色背景  │  Content Area             │
│          │  （表格/图表/表单）         │
└──────────┴──────────────────────────┘
```

Dashboard 强调数据密度——财务数据紧密排列，但 UI chrome 间隔充足，创造"有序密度"的感觉。

## 二、Grid 系统

### 页面级 Grid

```
Content Wrapper:   max-width: 1080px, mx-auto
Gutter Desktop:    32px
Gutter Tablet:     24px
Gutter Phone:      16px
Feature Grid:      grid-cols-2 或 grid-cols-3
Gap:               基于 8px 网格
Section Spacing:   96px (desktop) / 64px (tablet) / 40px (phone)
```

### Grid 规律

| 特征 | Stripe 做法 |
|---|---|
| **容器宽度** | 1080px（比常见的 1200px 略窄，更聚焦） |
| **列数** | 2-3 列（features）/ 单列（Hero、文案） |
| **间距** | 8px 基础网格，间距密集在小数值端 |
| **对齐** | 极精确——这是金融产品的基本要求 |
| **响应式** | 3 列 → 2 列 → 单列堆叠 |

### 为什么感觉规整？

1. **间距系统在小数值端密集**——1px, 2px, 4px, 6px, 8px... 每 2px 一个档位，反映金融数据 UI 对精度的需求
2. **1080px 容器**——比标准的 1200px 窄，让内容更紧凑聚焦
3. **深色 section 全宽**——打破容器约束，创造沉浸式品牌体验

## 三、Typography

### 字体家族

| 用途 | 字体 | 说明 |
|---|---|---|
| **Display / Body / UI** | Söhne（sohne-var） | Stripe 定制可变字体，几何瑞士风格 |
| **Mono / Code** | Source Code Pro | Adobe 开源等宽字体 |
| **Fallback** | SF Pro Display, -apple-system, system-ui | 系统兜底 |

### 关键特征：`ss01` Stylistic Set

Stripe 的每个文本元素都启用 OpenType `"ss01"` 风格集。这会修改特定字形（可能是 `a`、`g`、`l` 等的替代形式），创造出 Stripe 标志性的几何、当代字母形态。**这是 Stripe 排版 DNA 的核心。**

### 字号体系

| 层级 | 字号 | 字重 | 行高 | 字间距 | 用途 |
|---|---|---|---|---|---|
| **Display Hero** | 56px | **300** | 1.03 | -1.4px | Hero 大标题 |
| **Display Large** | 48px | **300** | 1.15 | -0.96px | 次级 Hero |
| **Section Heading** | 32px | **300** | 1.10 | -0.64px | 区块标题 |
| **Sub-heading L** | 26px | **300** | 1.12 | -0.26px | 卡片标题 |
| **Sub-heading** | 22px | **300** | 1.10 | -0.22px | 小标题 |
| **Body Large** | 18px | **300** | 1.40 | normal | 导语 |
| **Body** | 16px | **300–400** | 1.40 | normal | 正文 |
| **Button** | 16px | **400** | 1.00 | normal | 按钮文字 |
| **Button Small** | 14px | **400** | 1.00 | normal | 紧凑按钮 |
| **Link** | 14px | **400** | 1.00 | normal | 导航链接 |
| **Caption** | 13px | **400** | normal | normal | 标签/元数据 |
| **Caption Small** | 12px | **300–400** | 1.33–1.45 | normal | 小字 |
| **Caption Tabular** | 12px | **300–400** | 1.33 | -0.36px | 财务数据（tnum） |
| **Micro** | 10px | **300** | 1.15 | 0.1px | 极小标签 |
| **Micro Tabular** | 10px | **300** | 1.15 | -0.3px | 图表数据（tnum） |
| **Nano** | 8px | **300** | 1.07 | normal | 最小标签 |
| **Code Body** | 12px | **500** | 2.00 | normal | 代码块 |
| **Code Bold** | 12px | **700** | 2.00 | normal | 代码关键字 |

### Typography 规律

| 规则 | 做法 | 为什么 |
|---|---|---|
| **Weight 300 是签名** | 标题和正文全部用 weight 300 | 用轻盈表达奢侈——不需要重量来获取权威 |
| **ss01 无处不在** | 所有 sohne-var 文本启用 `"ss01"` | 替代字形定义了品牌的几何感 |
| **渐进 tracking** | 字越大 tracking 越紧：-1.4px(56px) → normal(16px) | 大字需要更紧凑的字间距 |
| **tnum 不混用** | 财务数据表格用 `"tnum"`，正文数字用 `"ss01"` | 两种 OpenType 模式永不重叠 |
| **两种字重统治全局** | 300（内容）+ 400（UI/按钮），无 600-700 | 极简重量 = 极简噪音 |
| **无 Bold（sohne-var）** | sohne-var 不使用 600+ 字重 | 对比度由字号和颜色创造，而非字重 |

### 为什么舒服？

1. **Weight 300 的"耳语权威"**——大多数公司用 600-700 的粗体标题来抓眼球。Stripe 反其道而行——轻盈的字重传递的是自信和优雅，仿佛在说"我们不靠大声喊叫来获取你的注意"
2. **ss01 的几何精准**——替代字形让每一个字母都感觉被精心设计过
3. **深海军蓝 `#061b31` 而非纯黑**——文字颜色带有微妙的暖蓝底调，比 `#000` 舒适得多
4. **代码行高 2.00**——代码块的行高极为宽松，保证开发者阅读体验
5. **负 tracking 在大字上**——创造了字母之间"被精确计算过"的距离感

## 四、配色

### 核心理念：感知均匀的紫蓝体系

Stripe 的配色建立在**感知均匀色彩空间（CIELAB）**之上。他们的色彩系统不是凭感觉调的，而是用自建工具在 Lab 色彩空间中可视化调整，确保每种颜色在相同数值下有相同的感知对比度。

### 主色体系

| Token | 色值 | 用途 |
|---|---|---|
| **Page Background** | `#ffffff` | 页面底色 |
| **Surface Warm** | `#f6f9fc` | 暖调表面（极微妙） |
| **Brand Dark** | `#1c1e54` | 深色沉浸区域 |
| **Deep Navy** | `#061b31` | 标题文字（不是纯黑！） |
| **Label Dark Slate** | `#273951` | 标签、次级标题 |
| **Body Slate** | `#64748d` | 正文、描述 |
| **Border Soft Blue** | `#e5edf5` | 默认边框 |

### 强调色

| Token | 色值 | 用途 |
|---|---|---|
| **Stripe Purple** | `#533afd` | **唯一的交互色**——CTA、链接、选中态 |
| **Purple Hover** | `#4434d4` | Hover 态 |
| **Purple Deep** | `#2e2b8c` | 图标 Hover |
| **Purple Light** | `#b9b9f9` | 弱背景 |
| **Purple Mid** | `#665efd` | 范围选择器 |

### 语义色

| Token | 色值 | 用途 |
|---|---|---|
| **Success Green** | `#15be53` | 成功状态 |
| **Success Text** | `#108c3d` | 成功文本 |
| **Warning Lemon** | `#9b6829` | 警告 |
| **Danger Ruby** | `#ea2261` | 破坏性操作（仅用于图标和装饰） |
| **Magenta** | `#f96bee` | 渐变装饰（不用于 UI 交互） |
| **Magenta Light** | `#ffd7ef` | Magenta 主题表面 |

### CIELAB 感知均匀色彩系统

Stripe 在 2019 年发表了里程碑式的博客文章《Designing accessible color systems》，详细解释了他们如何用**感知均匀色彩空间（CIELAB）**重建整个色彩系统：

- **问题**：传统 HSL 中，同样的 Lightness 值下，黄色看起来比蓝色亮得多
- **解决**：在 CIELAB 空间中调整颜色，确保每个颜色的感知亮度曲线一致
- **结果**：9 级颜色阶梯中，每一级的所有色调都有相同的对比度值
- **规则**：任意两个颜色相差 ≥5 级 ⇒ 小文本对比度达标；≥4 级 ⇒ 图标/大文本达标

### 颜色数量

| 类别 | 数量 |
|---|---|
| **中性色** | ~6 级（White → Surface Warm → Border → Slate → Dark Slate → Navy） |
| **主强调色** | **1 个（Purple）** + 5 级深浅变体 |
| **语义色** | Green / Lemon / Ruby / Magenta（各 2-3 级） |
| **同一页面实际彩色** | ≤ 3 种 |

### 为什么看起来高级？

1. **只有一种交互彩色**——紫色是 Stripe 唯一的链接和 CTA 颜色。不是"调色板"，是"信号"
2. **标题是海军蓝而非纯黑**——`#061b31` 带有微妙的蓝色底调，比 `#000` 更温暖、更有深度
3. **阴影也是品牌色的**——`rgba(50,50,93,0.25)` 是蓝紫色调的阴影，不是中性灰
4. **CIELAB 的数学精确性**——每种颜色在同一级别有相同的感知亮度。用户看不到数学，但能感受到秩序
5. **深色 section 用品牌紫蓝 `#1c1e54`**——不是黑、不是灰，是专属于 Stripe 的"暗色"

2. **Ghost 边框也是品牌色的**——`#b9b9f9` 淡紫边框让每个按钮都在传递品牌
3. **绝对不给 Primary 提供多种颜色变体**——只有一个紫色

## 六、Card

### 卡片体系（Stripe 的标志性设计）

Stripe 的卡片不使用简单的扁平或 Material 风格，而是使用**蓝调多层阴影系统**——这是 Stripe 最具辨识度的视觉元素之一。

| 属性 | 值 |
|---|---|
| **背景** | `#ffffff` |
| **边框** | `1px solid #e5edf5`（默认） |
| **圆角** | 4px–8px（保守范围） |
| **阴影** | **蓝调多层阴影**（见下文） |

### 标志性阴影系统

Stripe 的阴影不是灰色，而是**品牌色的延伸**：

| 级别 | 阴影值 | 用途 |
|---|---|---|
| **Flat (L0)** | none | 页面背景 |
| **Ambient (L1)** | `rgba(23,23,23,0.06) 0px 3px 6px` | 微妙抬升 |
| **Standard (L2)** | `rgba(23,23,23,0.08) 0px 15px 35px` | 标准卡片 |
| **Elevated (L3)** | `rgba(50,50,93,0.25) 0px 30px 45px -30px, rgba(0,0,0,0.1) 0px 18px 36px -18px` | 特色卡片、Dropdown |
| **Deep (L4)** | `rgba(3,3,39,0.25) 0px 14px 21px -14px, rgba(0,0,0,0.1) 0px 8px 17px -8px` | Modal |
| **Ring** | `2px solid #533afd` + 扩散光晕 | 键盘 Focus |

### 阴影哲学

1. **两层阴影 = 视差深度**：蓝色远层（大偏移、大模糊）+ 黑色近层（小偏移、小模糊），创造出类似视差的深度感
2. **负 spread 值**：`-30px, -18px` 确保阴影不超出元素水平范围，抬升感是纯垂直的
3. **蓝色调 `rgba(50,50,93)`**：阴影本身的颜色来自品牌海军蓝，不是中性灰——连抬升都在传递品牌

### 卡片设计规律

1. **蓝调阴影是 Stripe 的视觉签名**——没有第二个公司用这种阴影配方
2. **圆角范围 4-8px**——不做大圆角，保持金融产品的严肃感
3. **深色 section 中的卡片**使用 `rgba(255,255,255,0.1)` 半透明边框
4. **Hover 时阴影强化**——增加蓝调层的强度

## 八、Animation

### 动画哲学

Stripe 的动画哲学是**功能性优先**——动画服务于信息传达，而非装饰。

| 规则 | 做法 |
|---|---|
| **快速** | `--motion-fast: 150ms`, `--motion-base: 200ms` |
| **Easing** | `cubic-bezier(0.2, 0, 0, 1)`（标准 ease-out） |
| **用途** | Hover 反馈、卡片抬升、页面切换、数据可视化 |
| **不堆砌** | 没有 Scroll-triggered 叙事动画，没有视差滚动 |

### 微交互清单

| 场景 | 动画 | 时长 | Easing |
|---|---|---|---|
| **按钮 Hover** | 背景色切换 | 150ms | standard |
| **卡片 Hover** | 阴影强化 + 微抬升 | 200ms | standard |
| **导航链接** | 底部下划线/颜色变化 | 150ms | standard |
| **Dropdown** | 展开 + 淡入 | 150–200ms | standard |
| **Modal** | 淡入 + 抬升 | 200ms | standard |
| **数据可视化** | 图表数值动画 | 持续 | — |
| **全局 GDP 计数器** | 实时数字滚动 | 持续 | — |

### 值得学习的点

1. **动画是反馈，不是表演**——150-200ms，不拖沓
2. **统一的 easing 曲线**——`cubic-bezier(0.2, 0, 0, 1)` 贯穿整个系统
3. **数据动画而非装饰动画**——Stripe 的"动"在于实时数据的更新，而非花哨的页面转场

## 九、为什么看起来高级

### 9.1 Weight 300：耳语权威

这是 Stripe 设计中最颠覆性的选择。**没有人用 300 字重做 56px Hero 标题**——所有人都在用 600-700。Stripe 用轻盈表达了奢侈：我们的信息如此重要，不需要喊叫。这不仅仅是审美，这是一种品牌态度。

### 9.2 蓝调阴影：品牌色的延伸

`rgba(50,50,93,0.25)` 不是随便选的灰色阴影。它是品牌海军蓝 `#061b31` 的同色调。当卡片从页面"抬起"时，投射的阴影是品牌色的——这在全世界找不到第二个公司这样做。**阴影不再是中性的深度工具，而是品牌叙事的载体。**

### 9.3 CIELAB：数学驱动的色彩系统

Stripe 不是凭感觉选颜色的。他们自建工具，在 CIELAB 感知均匀色彩空间中可视化地调整每一种颜色，确保：
- 每种色调在同一等级有相同的感知对比度
- 黄色和蓝色"看起来"一样亮
- 9 级阶梯可以直接计算对比度（差 5 级 = AA）

这是**工程思维应用于色彩设计**的极致体现。

### 9.4 单一交互色

整个网站只有一种可点击的颜色：紫色。链接、按钮、选中态——都是紫色的变体。Ruby（`#ea2261`）和 Magenta（`#f96bee`）只用于渐变装饰，绝不用于交互。这种**色彩纪律**让紫色每次出现都有明确含义。

### 9.5 ss01：字体的品牌化

`"ss01"` stylistic set 被应用于每个文本元素。这修改了 Söhne 字体的特定字形，创造出属于 Stripe 的独特字母形态。再加上 weight 300 + 负 tracking 的组合，形成了**一眼就能认出的 Stripe 排版签名**。

### 9.6 阴影的视差深度

双层阴影（蓝色远层 `50,50,93` + 黑色近层 `0,0,0`）创造了类似视差的深度感。这不是简单的 box-shadow——这是精心设计的"空间幻觉"。

### 9.7 深浅交替的建筑节奏

页面在白色 section 和深蓝色 section（`#1c1e54`）之间交替。这种明暗节奏创造了一种"呼吸感"——读者在看完一个白色的区块后，进入沉浸式的深色品牌空间，然后再回到白色。这不是随机的，是刻意编排的叙事节奏。

### 9.8 不做多余的事

- 不用多种彩色 → 用 1 个紫色
- 不用多种字重 → 只用 300 和 400
- 不用大圆角 → 4-8px 保守范围
- 不用长动画 → 150-200ms
- 不用装饰性动效 → 只动画数据和反馈

但 Stripe 在**别人不投入的地方投入**：
- 自建 CIELAB 色彩工具
- 定制可变字体 Söhne
- 蓝调多层阴影系统
- `"ss01"` 在每个文本元素上的极致贯彻

## 十、可纳入 Design System 的设计资产

### 可直接采纳的原则

| 原则 | 适用场景 |
|---|---|
| **蓝调阴影（品牌色阴影）** | 任何有品牌色的产品（用品牌色的暗调做阴影） |
| **单一交互色** | 所有类型产品 |
| **深浅交替区块节奏** | Landing Page |
| **Weight 300 作为标题字重** | 品牌定位偏高端/自信的产品 |
| **tnum + ss01 不重叠** | 有数据展示的产品 |
| **感知均匀色彩系统** | 有可访问性要求的复杂产品 |
| **双层阴影视差** | 卡片密集型产品 |
| **150-200ms 动画上限** | 全局 |

### 建议新增的 Design Token

```
颜色系统:
  - 标题色不用纯黑，用深海军蓝（#061b31 → #0a1e3d → #1a1a2e）
  - 阴影色用品牌色暗调，不用中性灰
  - 感知均匀的 9 级色彩阶梯
  - 单交互色 + 少量语义色（非装饰性）

间距系统:
  - 小数值端密集：1px, 2px, 4px, 6px, 8px...
  - 区块间距：96px (desktop) / 64px (tablet) / 40px (mobile)

阴影系统:
  - 蓝调多层：rgba(brand-r, brand-g, brand-b, 0.25) 远层 + rgba(0,0,0,0.1) 近层
  - 负 spread 值控制水平扩散

字体系统:
  - Weight 300 作为签名字重
  - ss01 风格集品牌化
  - tnum 专用于数据
```

### 建议新增的组件规范

- **Chromatic Shadow Card**：蓝调多层阴影卡片（品牌色阴影的通用化模式）
- **Dark Brand Section**：全宽深色沉浸区域
- **Tabular Data Display**：tnum + 等宽数字的数据展示模式
- **Perceptually Uniform Color Scale**：感知均匀的色彩阶梯（可复用的颜色生成方法）

### 建议更新的 Checklist

在 `checklists/ui-review.md` 中增加：
- [ ] 阴影是否使用品牌色暗调而非中性灰？
- [ ] 交互色是否只有一种？（能否减少？）
- [ ] 标题色是否为深海军蓝而非纯黑？
- [ ] 动画是否 ≤200ms？
